Here, we’ll be parsing our jupyter notebook from a random github link, and reading it out into markdown:

  1. Markdown = markdown
  2. Code = markdown code block
  3. Code output = Images/code block

Furthermore, we will fetch a github repo and automatically update it, when the file at the given like has been updated.

Latstly, we’ll be able to stack multiple notebooks into one ‘project’.

Running scripts on website#

Since our site is static, we’ll need to have a built-in link to displaying our github page, and automatically converting it to markdown somehow inbetween.

First, let’s do a test. We’ll load in the raw code from an arbitrary github file, and just change the first letter; to make sure we can handle changing, and loading dynamic content.

Modifying Markdown post#

Now that we can run our javascript, let’s make a change to the file that we’ve inserted the script on (any file with my_script: true).

Then we can simply call it with:

---
title: "Your Post Title"
date: '2024-12-19T16:39:50-05:00'
jsFiles:
  - "/js/remote-content.js"
---

We must include in our header-file now:

{{- if and .IsPage .Params.jsFiles -}}
    {{- range .Params.jsFiles -}}
        <script defer src="{{ . }}"> </script>
    {{- end -}}
{{- end -}}

To breifly explain this code, we are

  • checking if our page is actually a blog post
  • checking if we have js file params, and reading it.
  • adding the script to the header.

Program#

Now, we must actually write the program. Note that javascript only modifies the html file that is sitting on-top of the markdown file. For our purposes, we just change the markdown with a javascript function.

To understand how our markdown is loaded in an html document, we can look at our themes ‘single.html’ file, which is a mix of html and css.

We can see in our file, we have our content loaded as :

  {{- if .Content }}
  <div class="post-content">
    {{- if not (.Param "disableAnchoredHeadings") }}
    {{- partial "anchored_headings.html" .Content -}}
    {{- else }}{{ .Content }}{{ end }}
  </div>
  {{- end }}
